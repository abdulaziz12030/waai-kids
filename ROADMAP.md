# خارطة طريق مشروع نماء

هذه الوثيقة هي المرجع الأساسي المعتمد لبناء مشروع **نماء** وتطويره. أي تطوير جديد يجب أن يراعي هذه المراحل وألا يغيّر الأساس إلا بقرار واضح ومراجعة مسبقة.

## المرحلة 1: تأسيس المشروع تقنيًا

- إنشاء المشروع باستخدام Next.js وTypeScript.
- ربط المشروع بـ Supabase.
- النشر على Vercel.
- دعم اللغة العربية واتجاه RTL.
- تصميم متجاوب للجوال والآيباد والكمبيوتر.
- تجهيز بنية قابلة للتحول لاحقًا إلى PWA وتطبيق.
- حفظ مفاتيح Supabase داخل متغيرات البيئة في Vercel وعدم وضع المفاتيح الحساسة داخل GitHub.

## المرحلة 2: الموقع التعريفي

- صفحة رئيسية احترافية.
- شرح فكرة نماء.
- توضيح فوائد المنصة لولي الأمر والطفل.
- عرض مزايا القرآن والحفظ والأهداف والمهام والمكافآت.
- عرض الباقات والأسعار مبدئيًا دون تفعيل الدفع.
- توفير روابط تسجيل الدخول وإنشاء الحساب.

## المرحلة 3: تسجيل ولي الأمر

- إنشاء حساب بالبريد الإلكتروني.
- تسجيل الدخول والخروج.
- استعادة كلمة المرور.
- إنشاء أسرة.
- إضافة الأبناء.
- تسجيل بيانات الطفل الأساسية مثل الاسم والعمر والصف الدراسي والصورة الرمزية.
- يبقى حساب الطفل تابعًا لولي الأمر في هذه المرحلة.

## المرحلة 4: لوحة الأسرة

- عرض الأبناء.
- عرض مجموع النقاط.
- عرض الطلبات الجديدة.
- عرض الأهداف النشطة.
- عرض المهام المنتظرة.
- عرض المكافآت المستحقة.
- عرض تقدم الحفظ القرآني.
- عرض تقرير أسبوعي مختصر.
- إنشاء بطاقة مستقلة لكل ابن تشمل نقاطه وهدفه الحالي وآخر مهمة وآخر إنجاز قرآني.

## المرحلة 5: نظام الأهداف

- تمكين الطفل أو ولي الأمر من إنشاء هدف.
- دعم الأهداف المالية والعينية والتعليمية والسلوكية.
- تمكين ولي الأمر من الموافقة أو الرفض أو تحويل الهدف إلى خطة إنجاز.
- دعم مشاركة الطفل بجزء من قيمة الهدف.
- ربط الهدف بالنقاط أو المهام.
- تحديد قيمة الهدف والمدة والنقاط المطلوبة وتاريخ الاستحقاق.

## المرحلة 6: المهام والنقاط

- إنشاء مهام سلوكية وتعليمية وأسرية ودينية.
- تحديد عدد النقاط لكل مهمة.
- تحديد تاريخ البداية والنهاية.
- دعم التكرار اليومي أو الأسبوعي.
- اعتماد المهمة من ولي الأمر.
- حالات المهمة: مطلوبة، تم تنفيذها، بانتظار الاعتماد، معتمدة، مرفوضة.
- إضافة النقاط تلقائيًا بعد الاعتماد.

## المرحلة 7: نظام القرآن

- تحديد السورة وبداية الآية ونهايتها.
- دعم نصف وجه أو وجه كامل أو عدد آيات محدد.
- دعم الحفظ الجديد والمراجعة القريبة والمراجعة البعيدة.
- حالات الإنجاز: تم التكليف، تم الحفظ، تم التسميع، تم الإتقان، يحتاج مراجعة.
- إضافة نص الآيات بالرسم العثماني من مصدر موثوق في مرحلة مناسبة.
- فصل بيانات المصحف عن بيانات المستخدمين.

## المرحلة 8: المكافآت والاستحقاقات

- تحويل الهدف المكتمل إلى مكافأة مستحقة.
- دعم الصرف الكامل أو الجزئي.
- دعم التأجيل وطلب المهلة.
- دعم استبدال المكافأة أو تحديد موعد جديد.
- إظهار موعد التسليم للطفل.
- حفظ سجل واضح للمكافآت والمستحقات المؤجلة.

## المرحلة 9: حساب الطفل

- إنشاء دخول مبسط للطفل.
- استخدام رمز عائلي ورقم تعريف ورمز سري بسيط.
- تمكين الطفل من رؤية أهدافه ومهامه ونقاطه وخطط القرآن ومكافآته.
- منع الطفل من الاطلاع على بيانات ولي الأمر أو الإعدادات المالية.

## المرحلة 10: التقارير

- تقارير أسبوعية وشهرية.
- مجموع النقاط.
- المهام المكتملة.
- نسبة الالتزام.
- الأهداف المحققة.
- عدد صفحات القرآن المحفوظة.
- المكافآت المصروفة والمؤجلة.
- دعم تنزيل التقارير بصيغة PDF لاحقًا.

## المرحلة 11: الاشتراكات

- إضافة نظام الدفع بعد استقرار الوظائف الأساسية.
- الباقة الأسرية المقترحة: 19 ريال شهريًا أو 149 ريال سنويًا.
- تشمل حتى 6 أبناء وولي أمر إضافي ومزايا القرآن والأهداف والمكافآت.
- إمكانية توفير تجربة مجانية لمدة 14 يومًا.
- عدم تعقيد النسخة الأولى باشتراكات المعلمين والمدارس.

## المرحلة 12: المعلم المستقل

- إنشاء حساب معلم.
- دعوة الطلاب برمز.
- ربط الطالب نفسه بولي أمره ومعلمه دون إنشاء حساب مكرر.
- اعتماد الحفظ والتسميع.
- إضافة ملاحظات وخطط جماعية.
- منع المعلم من الاطلاع على الأموال أو مكافآت الأسرة أو البيانات الخاصة.

## المرحلة 13: الحلقات والمدارس

- إضافة مدير الحلقة أو المدرسة.
- إضافة المشرفين والمعلمين.
- إنشاء المجموعات والفصول.
- إدارة الطلاب والصلاحيات.
- إنشاء تقارير جماعية.
- خطط اشتراك حسب عدد الطلاب والمعلمين.
- البناء فوق قاعدة المشروع الحالية دون إنشاء نظام منفصل.

- يتم تنفيذ المشروع تدريجيًا وفق ترتيب الأولويات.
